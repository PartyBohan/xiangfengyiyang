# 音乐密码 · 一首歌四关练习

围绕一首歌生成四个连续 PartyKeys 练习关卡的网页原型：

1. 完整示范聆听。
2. 卡拉 OK 式和弦跟弹。
3. 根音、五音、右手和弦与琶音织体。
4. 左手伴奏加右手旋律的完整演奏。

## 当前示范

- 曲目：《像风一样》教学占位编配。
- 网络资料核对到原曲常见为 D 调、约 59 BPM 慢拍；首版按需求使用 C–Am–F–G，并整体下移一个全音。
- 由于正式 MusicXML 尚未提供，当前旋律、节奏和歌词均明确标记为占位内容，不冒充原曲正式曲谱。
- 页面可直接上传 `.xml` 或 `.musicxml`，在浏览器内解析标题、速度、拍号、调号、旋律、和弦和歌词，并重新生成四关。

## 输入、音色与灯光

- 支持 Web MIDI / MidiBrowser 输入，也支持屏幕虚拟键盘。
- 普通 MIDI 键盘可发声与判定；只有识别为 PartyKeys 的输出才发送专用 SysEx。
- PartyKeys 36 使用 CMD `0x15` RGB、连接后 `0F 01` 初始化、差异更新、停止时全灭；单帧不超过 256 bytes。
- 内置 Salamander Grand Piano V3 的 13 音高 × 4 力度采样，加载前使用合成音色兜底。署名见 [AUDIO_CREDITS.md](AUDIO_CREDITS.md)。

## 36 / 72 键模式

- 单琴：MIDI 48–83（C3–B5）。
- 双琴原型：第一台 PartyKeys 映射为 MIDI 36–71（C2–B4），第二台映射为 72–107（C5–B7）。两台琴仍分别接收各自 0–35 的灯光 key index。
- 双琴顺序目前按浏览器枚举到的 PartyKeys 输入顺序决定。正式发布前需要在两台真实 PartyKeys 上验证顺序、重连、灯光延迟和设备区分，并补充可视化的“左琴/右琴”分配界面。

## 本地运行

```bash
npm install
npm run dev
npm run build
```

真实硬件、BLE MIDI 与双琴 72 键映射尚未在本项目中完成真机验收，不能视为发布就绪结论。
